Milestone

Today was awesome and managed to glue all the code together, bearing in mind I've had family over today.

Since the last blog entry AJAX is now fixed

As it stands, I now have the following process working

  1. HTML5, Javascript request through AJAX
  2. AJAX request to PHP
  3. PHP executes the C file with the command './serial getData 0 > log.txt 2>error.txt'
  4. C file pipes output to the log file
  5. PHP reads the file and turns it to one long string variable
  6. PHP then "explodes" the string using a semi colon to seperate the sensors and values.
  7. PHP then creates to substrings one with the sensor name, the other with the sensor value, this is seperated by a colon.
  8. More filtering and processing is done in PHP.
  9. PHP sends the sensor data as an array using AJAX
  10. Javascript receives the array using AJAX and appends the sensor value to the corresponding position in the user interface as document IDs
  11. Sensors update every 8 seconds on the user interface

I currently have the following sensors implemented in the user interface

  • Ultrasonics
  • RFID and rock samples
  • Temperature Humidity
  • Battery Voltage

The sensors left are the camera with filters, correct Battery voltage and IMU data, as well as a demo button for autonomous control recommended by Laurence.

Agile Methodologies

Tonight I have spent some time trying to decide what type of Agile Methodology would be suitable for my Major Project. I have also spent some time trying out different open source software for managing tasks and time.

Kanban is based on three principles:

  1. Visualize what you do today (workflow): seeing all the items in context of each other can be very informative
  2. Limit the amount of work in progress (WIP): this helps balance the flow-based approach so teams don't start and commit to too much work at once
  3. Enhance flow: when something is finished, the next highest thing from the backlog is pulled into play

What makes Kanban a suitable methodology for my project?

  • No prescribed roles
  • Continuous Delivery
  • Work is pulled through the system (single piece flow)
  • Changes can be made at any time
  • Cycle time
  • More appropriate in operational environments with a high degree of variability in priority

The benefits of using Kanban are:

  • Shorter cycle times can deliver features faster.
  • Responsiveness to Change:
  • When priorities change very frequently, Kanban is ideal.
  • Balancing demand against throughput guarantees that most the customer-centric features are always being worked.
  • Requires fewer organization / room set-up changes to get started
  • Reducing waste and removing activities that don't add value to the team/department/organization
  • Rapid feedback loops improve the chances of more motivated, empowered and higher-performing team members

Information on Kanban at https://www.versionone.com/what-is-kanban/
A rather useful download on kanban can be found here

Problems I may encounter along the way with Kanban:

  • Sticking to the WIP limit
    • The difficulty of a task underestimated (overshoot the WIP limit)
    • Changes to the way the software works (eating into WIP cycle)
